| 班级管理 | 班级、班级学生、归档 | `class_worker` |

## 5. UI 设计引用规范

UI 以 `assets\ui-reference\img` 的 EduPro 风格为主。

### 5.1 视觉风格

| 项 | 规范 |
| --- | --- |
| 整体气质 | 清爽、教育科技、轻量游戏化 |
| 主色 | 蓝紫色系，参考 `#587CFF` |
| 辅助色 | 绿色 `#22C55E`、橙色 `#FFB020`、粉色 `#FF5BAA`、灰蓝 `#64748B` |
| 背景 | 浅灰白背景，避免大面积深色 |
| 卡片 | 小圆角、轻阴影、信息密度适中 |
| 图标 | 使用统一线性图标，按钮优先 icon + tooltip |
| 插画 | 少年教育、编程、游戏化、作品社区相关 |
| 字体 | 中文优先思源黑体，英文 Source Han Sans 或系统无衬线 |

### 5.2 页面参考

| 端 | 页面 | UI 要点 |
| --- | --- | --- |
| 管理后台 | 首页仪表盘 | 左侧深色导航，顶部工具栏，统计卡片，折线图、环形图、待办列表 |
| 教师端 | 我的班级、课堂点名 | 卡片式班级列表，课次进度，二维码/签到入口，学生签到状态 |
| 学生端 | 今日课程、积分、作品社区、测评 | 移动端优先，任务卡片，积分卡片，作品列表，底部导航 |
| 家长端 | 学习报告 | 学生信息、学习能力雷达图、测评趋势、学习数据摘要 |

### 5.3 开发时注意

- 不要做营销式首页，第一屏直接进入可操作系统。
- 后台页面保持高信息密度，适合教务反复使用。
- 移动端学生页可以更活泼，但不要影响任务入口。
- 所有表格都要有搜索、筛选、分页、空状态和加载状态。
- 危险操作必须二次确认，例如删除、停用、恢复备份、重置密码。

## 6. 角色权限清单

权限必须以后端校验为准，前端隐藏菜单只能作为体验优化。

### 6.1 角色定义

| 角色 | 定位 |
| --- | --- |
| 超级管理员 | 系统最高权限，初始化、账号权限、全部业务、备份恢复 |
| 校长 | 校区负责人，查看和管理校区业务，不恢复数据库、不分配超级权限 |
| 学管师 | 教学运营，学生、班级、排课、签到、统计、课程资源 |
| 行政 | 前台支持，学生资料、班级名单、签到协助、统计查看 |
| 教师 | 授课人员，只能查看和维护本人相关班级、课次、签到、课程详情 |
| 学生 | 查看本人课程、任务、作品、积分，提交作品 |
| 只读账号 | 仅查看，不允许写入 |

### 6.3 权限开发硬规则

- 每个 API 都必须绑定权限点。
- 所有写操作必须检查角色和数据范围。
- 教师不能访问其他教师的班级、课次、签到和学生作品。
- 学生只能访问自己的作品、任务、积分和学习报告。
- 只读账号拦截所有非 `GET/HEAD/OPTIONS` 请求。
- 管理员重置密码、分配角色、恢复备份、删除数据必须写操作日志。
- 批量导入、模板导入、素材上传必须记录导入人和来源。

## 7. 开发优先级

### P0：系统入口和权限

- 统一登录认证。
- 超级管理员初始化。
- 角色权限中间件。
- 当前用户 `/api/me`。
- 操作日志基础。

### P1：教务基础闭环

- 学生、教师、班级。
- 预设课程库。
- 课次排课和批量生成。
- 签到扣课。
- 每节课详情。

### P2：Scratch 课堂闭环

- `.sb3` 模板导入。
- 课次绑定模板。
- 学生复制模板为个人作品。
- 保存作品和缩略图。
- 提交作品。
- 教师查看和点评。

### P3：素材库和作品展示

- Scratch 素材分类。
- 素材上传、审核、启停。
- 作品发布、开源、点赞、收藏。
- 优秀作品推荐。

### P4：积分和扩展

- 接入积分流水。
- 签到、提交、推荐触发积分事件。
- 后续扩展拼豆、3D 打印、创客课程。

## 8. 开发前检查清单

开始编码前先确认：

- 当前主系统选择 `class_worker` 还是新建工程。
- 数据库采用 SQLite 继续迭代，还是迁移 MySQL/PostgreSQL。
- Scratch 能力直接接 `hydro_scratch`，还是先做轻量本地集成。
- UI 使用 `img` 参考图的 EduPro 风格。
- 角色权限矩阵先固化，不边写边补。
- 文件上传目录和素材访问策略先确定。
- 操作日志从第一版就接入，不后补。
