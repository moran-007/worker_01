## 四、人工页面测试建议

### 超级管理员

1. 登录 `qa_super_admin`。
2. 进入“账号权限”，确认可新增、编辑账号，并能看到角色权限预览。
3. 进入“日志”和“数据库”，确认页面可打开。
4. 备用入口进入“数据备份”，确认可看到恢复入口。

### 校长

1. 登录 `qa_principal`。
2. 确认可进入学生、教师、班级、课程、课次、统计、日志。
3. 确认看不到“账号权限”和“数据库”入口。
4. 尝试直接访问 `/users` 或 `/api/users`，应被拒绝。

### 学管师

1. 登录 `qa_academic`。
2. 确认可维护学生、班级、预设课程、课次、签到、课程详情。
3. 确认不可进入账号权限、日志、数据库。
4. 教师管理只应查看，不应新增编辑教师。

### 行政

1. 登录 `qa_office`。
2. 确认可查看学生、教师、班级、课程、课次、统计。
3. 确认可维护学生和班级名单。
4. 确认不可新增教师、不可维护课程类型、不可进入日志和数据库。

### 教师

1. 登录 `qa_teacher`。
2. 确认只能查看绑定教师 `moran` 相关课次和班级范围。
3. 确认可进入本人课次的签到和课程详情。
4. 确认不可新增学生、教师、课程类型，不可进入账号权限、日志、数据库。

### 只读账号

1. 登录 `qa_readonly`。
2. 确认可浏览工作台、学生、教师、班级、课程、课次、统计。
3. 确认页面上不出现新增、编辑、导入、保存类按钮。
4. 直接提交任意 POST 接口应返回 403。

## 五、后续维护

1. 测试账号仅用于权限验证，上线前请停用或修改密码。
2. 新增角色或模块时，先更新 `permissions.py` 的权限点和角色矩阵。
3. 新增页面或 API 时，必须同步补充端点权限映射和前端路由权限。
4. 如果后续启用总部、财务、课程顾问，应先补齐对应业务模块，再开放账号分配。
