下面给你整理成一套**少儿编程/教育培训机构适用的完整教务管理系统功能需求**，偏向你个人或机构后续开发时可以直接拿来做需求文档。结合你之前课程课件、班级、课时、学生管理这类场景，系统应重点围绕“招生—报名—排课—上课—消课—续费—统计”闭环来设计。

# 教务管理系统功能模块补充方案

## 一、基础数据管理

### 1. 学生管理

用于管理所有在读、试听、停课、结课、退费学生。

可包含功能：

1. 学生姓名、性别、年龄、生日、学校、年级。
2. 家长姓名、手机号、微信号、紧急联系人。
3. 学生状态：潜在学员、试听学员、正式学员、停课学员、结课学员、退费学员。
4. 学生标签：Scratch、Python、C++、机器人、竞赛班、考级班、低龄启蒙等。
5. 学习备注：性格特点、学习能力、注意事项、过敏/身体情况等。
6. 成长档案：上课记录、作业记录、比赛记录、考级记录、作品记录。

### 2. 家长管理

如果后期学员多，建议把家长单独做成模块。

可包含功能：

1. 一个家长可绑定多个孩子。
2. 家长联系方式管理。
3. 家长沟通记录。
4. 家长意向等级：高、中、低。
5. 家长来源：转介绍、地推、朋友圈、抖音、小红书、门店咨询、老生续费。
6. 家长投诉、建议、回访记录。

### 3. 教师管理

用于管理授课老师、助教、班主任、课程顾问。

可包含功能：

1. 教师基本信息。
2. 教师负责课程类型。
3. 教师可授课时间段。
4. 教师课时统计。
5. 教师课酬统计。
6. 教师请假、调课记录。
7. 教师评价记录。
8. 教师权限绑定。

### 4. 校区/教室管理

适合有多个教室或多个校区的机构。

可包含功能：

1. 校区名称、地址、负责人。
2. 教室名称、容量。
3. 教室设备：电脑、机器人、积木套装、投影仪等。
4. 教室占用状态。
5. 教室排课冲突提醒。

## 二、课程体系管理

### 1. 课程类型管理

比如少儿编程机构常见课程类型：

1. Scratch 图形化编程。
2. ScratchJr 幼儿编程。
3. Python 编程。
4. C++ 算法。
5. 机器人搭建。
6. Arduino 创客。
7. 信息素养竞赛。
8. WRC/RSC/NOC/蓝桥杯竞赛课程。
9. GESP/CSP-J 考级课程。
10. AI 启蒙课程。
11. 试听课。
12. 寒暑假集训课。
13. 一对一课程。
14. 小班课。
15. 大班公开课。

### 2. 课程管理

管理具体课程产品。

可包含功能：

1. 课程名称。
2. 课程类型。
3. 适合年龄段。
4. 适合基础：零基础、进阶、竞赛、考级。
5. 总课时数。
6. 单次课时长。
7. 课程价格。
8. 课程简介。
9. 课程目标。
10. 课程大纲。
11. 配套教材/教具。
12. 是否启用。
13. 是否允许试听。

### 3. 课程包管理

用于销售课程套餐。

例如：

1. 24课时 Scratch 入门包。
2. 48课时 Python 进阶包。
3. 96课时 C++ 算法长期班。
4. 10次机器人体验包。
5. 寒假集训 12课时包。
6. 竞赛冲刺 20课时包。

可包含功能：

1. 课程包名称。
2. 课时数量。
3. 售价。
4. 有效期。
5. 适用课程。
6. 是否允许赠课。
7. 是否允许退费。
8. 是否允许转课。
9. 优惠规则。

## 三、班级管理

### 1. 班级创建

可包含功能：

1. 班级名称。
2. 所属课程。
3. 授课老师。
4. 助教/班主任。
5. 上课教室。
6. 上课时间段。
7. 班级容量。
8. 当前人数。
9. 开班日期。
10. 预计结课日期。
11. 班级状态：未开班、进行中、已结课、暂停、取消。

### 2. 班级学生管理

可包含功能：

1. 添加学生到班级。
2. 从班级移除学生。
3. 学生转班。
4. 学生插班。
5. 学生退班。
6. 班级名单导出。
7. 班级花名册打印。
8. 班级学生剩余课时显示。

### 3. 班级课表管理

可包含功能：

1. 固定周期排课，例如每周六 10:00–11:30。
2. 临时加课。
3. 停课。
4. 调课。
5. 补课。
6. 节假日自动避开。
7. 排课冲突检测：教师冲突、教室冲突、学生冲突。
8. 一键生成整期课表。

---

## 四、排课与课时管理

### 1. 排课管理

这是教务系统核心模块之一。

可包含功能：

1. 按班级排课。
2. 按老师排课。
3. 按教室排课。
4. 按学生排课。
5. 支持单次课、循环课、临时课。
6. 支持拖拽调整课表。
7. 支持日历视图、周视图、月视图。
8. 支持课程冲突提醒。
9. 支持取消课程并记录原因。
10. 支持课程备注。

### 2. 签到管理

用于上课消课。

可包含功能：

1. 老师手动签到。
2. 学生状态：到课、请假、旷课、迟到、早退、补课。
3. 到课自动扣课时。
4. 请假不扣课时。
5. 旷课是否扣课时可配置。
6. 补课记录。
7. 签到备注。
8. 按日、周、月统计到课情况。
9. 导出签到表。

### 3. 课时管理

用于记录学生购买、消耗、剩余课时。

可包含功能：

1. 购买课时。
2. 赠送课时。
3. 消耗课时。
4. 冻结课时。
5. 延期课时。
6. 转让课时。
7. 退费扣减课时。
8. 剩余课时预警。
9. 课时即将到期提醒。
10. 课时明细流水。

建议每一条课时流水都记录：

1. 变动时间。
2. 变动类型。
3. 增加或减少数量。
4. 关联课程/订单/班级。
5. 操作人。
6. 备注。

---

## 五、订单与收费管理

### 1. 订单管理

用于记录学生报名、续费、转课、退费等业务。

可包含功能：

1. 新建订单。
2. 选择学生。
3. 选择课程包。
4. 应收金额。
5. 实收金额。
6. 优惠金额。
7. 支付方式：微信、支付宝、现金、银行卡、对公转账。
8. 支付状态：未支付、部分支付、已支付、已退款。
9. 经办人。
10. 订单备注。
11. 合同/收据上传。

### 2. 续费管理

可包含功能：

1. 剩余课时低于设定值提醒。
2. 距离到期日期提醒。
3. 续费记录。
4. 续费转化率统计。
5. 班主任跟进记录。
6. 老学员续费优惠规则。

### 3. 退费管理

培训机构实际很常用，建议单独设计。

可包含功能：

1. 发起退费申请。
2. 查看已上课时。
3. 查看剩余课时。
4. 自动计算可退金额。
5. 扣除教材费、活动费、优惠差额等。
6. 退费审核。
7. 退费完成记录。
8. 退费原因分类。
9. 退费数据统计。

### 4. 优惠与活动管理

可包含功能：

1. 优惠券。
2. 满减活动。
3. 老带新奖励。
4. 团报优惠。
5. 续费优惠。
6. 试听转正优惠。
7. 赠课规则。
8. 活动有效期。

## 六、权限管理

### 1. 账号管理

可包含功能：

1. 创建账号。
2. 禁用账号。
3. 修改密码。
4. 重置密码。
5. 登录日志。
6. 操作日志。

建议设置以下角色：

1. 超级管理员。
2. 校长/负责人。
3. 教务主管。
4. 任课老师。
5. 助教。
6. 班主任。
7. 课程顾问。
8. 财务。
9. 前台。
10. 只读查看账号。

### 3. 权限控制

可按模块控制：

1. 学生管理权限。
2. 课程管理权限。
3. 班级管理权限。
4. 排课权限。
5. 签到权限。
6. 订单权限。
7. 财务查看权限。
8. 退费审批权限。
9. 数据导出权限。

1. 查看。
2. 新增。

4. 删除。
5. 导出。
6. 审核。
7. 作废。

## 七、教学过程管理

### 1. 上课记录

每节课结束后老师填写。

可包含功能：

1. 本节课主题。
2. 本节课知识点。
3. 学生课堂表现。
4. 完成作品。
5. 存在问题。
6. 下节课安排。
7. 是否布置作业。
8. 是否需要家长沟通。

### 2. 作业管理

可包含功能：

1. 老师发布作业。
2. 学生提交作业。
3. 老师批改作业。
4. 作业评分。
5. 作业点评。
6. 未交作业提醒。
7. 优秀作业展示。
8. 作品附件上传，例如 Scratch 文件、截图、视频、代码文件。

### 3. 学习成长档案

可包含功能：

1. 学生课程进度。
2. 学生作品记录。
3. 学生课堂照片。
4. 学生比赛获奖记录。
5. 学生考级记录。
6. 老师评语。
7. 阶段测评报告。
8. 学习能力标签：逻辑思维、创造力、专注力、表达能力、动手能力。

### 4. 测评管理

可包含功能：

1. 入学测评。
2. 阶段测评。
3. 结课测评。
4. 考级模拟测评。
5. 竞赛模拟测评。
6. 测评报告生成。
7. 测评分数统计。
8. 薄弱知识点分析。

# 建议优先开发版本

如果你是想先做一个实用版本，不建议一开始做太大。可以分成三个阶段。

## 第一阶段：基础可用版

先做这些就够机构日常运转：

1. 学生管理。
2. 家长管理。
3. 课程管理。
4. 班级管理。
5. 排课管理。
6. 签到消课。
7. 课时余额。
8. 订单收费。
9. 数据导出。
10. 权限管理。

## 第二阶段：机构运营版

在基础版上增加：

# 最核心的数据逻辑

这个系统最重要的逻辑是：

学生报名购买课程包，生成订单；订单支付成功后，给学生增加对应课时；学生加入班级后，系统生成课表；每次上课签到后自动扣除课时；剩余课时不足时提醒续费；所有订单、课时、签到、排课、教师授课数据都进入统计报表。

也就是：

**学生 → 订单 → 课时 → 班级 → 排课 → 签到 → 消课 → 续费/退费 → 统计**

这个闭环做好，系统就真正有用了。





# 技术选型

建议你这个教务管理系统不要一上来搞太复杂，技术选型按“中小型教培机构管理后台”来做最合适。

我推荐两套方案：一套是**最稳妥版**，一套是**更现代、后期扩展更强版**。

## 方案一：推荐首选，Django 全栈管理系统

适合你现在这种需求：课程、班级、学生、订单、课时、权限、统计、导出。

### 技术栈

前端：

```text
Vue 3 + TypeScript + Element Plus + Pinia + Axios + ECharts
```

后端：

```text
Python + Django + Django REST Framework
```

数据库：

```text
PostgreSQL / MySQL
```

缓存与任务：

```text
Redis + Celery
```

部署：

```text
Docker + Nginx + Gunicorn
```

文件与导出：

```text
openpyxl / pandas / xlsxwriter
```

权限：

```text
Django 自带用户体系 + RBAC 角色权限
```

Django 的优势是后台管理、用户权限、ORM、表单、管理端这些能力非常成熟，适合快速开发这种数据管理型系统。Django 官方也将其定位为支持快速开发、结构清晰的高级 Python Web 框架。([GitHub][1])

前端用 Vue 3 + Element Plus 很适合做后台管理系统，Element Plus 本身就是面向 Vue 3 的组件库，表格、表单、弹窗、日期选择、分页、菜单这些后台系统常用组件比较齐全。([Element Plus][2]) Pinia 用来管理登录状态、用户信息、菜单权限、全局配置比较合适，官方说明它就是 Vue 的状态管理库。([Pinia][3])

### 适合原因

这套最适合你的项目，因为教务系统本质上是大量数据表之间的关系管理：

学生、家长、老师、课程、班级、订单、课时、签到、退费、作业、比赛、库存、权限。

Django 的 ORM 做这些关系表很顺手，而且后台管理页面可以快速生成，早期开发效率很高。

---

## 方案二：更现代，FastAPI 前后端分离

如果你希望系统后期对接小程序、App、家长端、AI 生成报告，推荐这个。

### 技术栈

前端：

```text
Vue 3 + TypeScript + Element Plus + Pinia + Axios + ECharts
```

后端：

```text
Python + FastAPI + SQLAlchemy / SQLModel + Pydantic
```

数据库：

```text
PostgreSQL
```

数据库迁移：

```text
Alembic
```

缓存与任务：

```text
Redis + Celery / Dramatiq
```

部署：

```text
Docker + Nginx + Uvicorn/Gunicorn
```

FastAPI 不强制绑定某一种数据库方案，可以配合 SQLAlchemy、SQLModel 使用，并支持 PostgreSQL、MySQL、SQLite 等常见关系数据库。([FastAPI][4]) Alembic 适合做数据库迁移管理，官方文档也明确提供迁移脚本生成、升级操作、离线 SQL 等能力。([Alembic][5])

### 适合原因

FastAPI 更适合做纯接口服务，接口文档自动生成，前后端分离体验好。后期如果你要加：

家长小程序、老师移动端、微信通知、AI 学习报告、外部支付接口、多校区 SaaS 化，FastAPI 会更舒服。

缺点是后台管理、权限、审计日志这些要自己搭更多，不如 Django 省事。

---

## 我更建议你选：Vue 3 + Django + PostgreSQL

你这个系统的重点不是高并发，而是：

数据关系复杂、权限多、表格多、导出多、统计多、后台操作多。

所以首选：

```text
前端：Vue 3 + TypeScript + Element Plus + Pinia + ECharts
后端：Django + Django REST Framework
数据库：PostgreSQL
缓存：Redis
任务：Celery
部署：Docker + Nginx
```

这套比较稳，适合先做成一个真正能用的产品。

建议：

适合页面：

## 2. 后端服务

建议：

```text
Django
Django REST Framework
Django SimpleJWT
Django Filter
Django CORS Headers
```

用途：

Django：核心业务系统。
Django REST Framework：提供 API 接口。
SimpleJWT：登录认证。
Django Filter：列表筛选，例如按班级、老师、时间、课程筛选。
CORS Headers：允许前后端分离访问。

适合模块：

学生管理、课程管理、班级管理、订单管理、课时管理、权限管理、签到管理、退费管理、报表管理。

---

## 3. 数据库

推荐：

```text
PostgreSQL
```

备选：

```text
MySQL
```

不建议正式版用 SQLite。

原因是正式教务系统会有大量关联查询，例如：

一个学生对应多个订单。
一个订单对应一个课程包。
一个课程包生成课时流水。
一个学生进入多个班级。
每节课要关联老师、教室、班级、学生签到记录。
每次消课又要关联课时流水。

PostgreSQL 更适合后期做复杂统计、JSON 字段、数据分析。

---

## 4. 缓存与异步任务

建议：

```text
Redis + Celery
```

用途：

1. 登录 Token 黑名单或验证码缓存。
2. 课时不足提醒。
3. 课程开始前提醒。
4. 生日提醒。
5. 续费提醒。
6. 批量导出 Excel。
7. 批量生成学习报告。
8. 定时统计日报、周报、月报。

早期可以不用 Celery，先把功能做出来。后期数据多了再加。

核心表：


角色可以设计为：

```text

校长
教务主管
老师
助教
课程顾问
财务
前台
只读账号
```

权限粒度建议做到：

```text
查看
新增
编辑
删除
导出
审核
作废
```

例如：

财务可以看订单和退费，但不能改学生上课记录。
老师可以看自己班级学生，但不能看全部财务数据。
课程顾问可以建试听和订单，但不能审批退费。
校长可以查看经营报表。
