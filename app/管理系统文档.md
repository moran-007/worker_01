# 课时签到与课时统计管理系统文档

> 更新时间：2026-05-17
>
> 当前系统已包含 Flask + SQLite 经典管理端、Vue3 + Element Plus 新前端、后端 API、数据库视图和 SQL 查询资产。

下面是一版较完整的《课时签到系统》功能需求说明，限定技术方向为 Python，本地运行或局域网部署。

【系统名称】

课时签到与课时统计管理系统

【使用场景】

适用于少儿编程、培训机构、兴趣班、小班课、线下课程，用于记录学生上课签到、教师授课、班级课时、课程类型，并支持按日、周、月统计和导出表格。

【部署要求】

系统应支持两种运行方式：

第一种是本地单机运行。数据保存在本机，适合一个前台或一个老师使用。

第二种是局域网部署。系统运行在一台电脑上，其他电脑、平板或手机在同一局域网内通过浏览器访问，例如访问：

```
http://192.168.1.100:8000
```

不依赖公网服务器，不需要购买云服务，适合校区内部使用。

技术上建议只使用 Python 生态：

后端：Python + Flask 或 FastAPI
数据库：SQLite，本地文件数据库即可
前端：HTML + CSS + JavaScript，也可以使用 Python 模板渲染
表格导出：openpyxl 或 pandas
打包运行：可选 PyInstaller，方便生成本地可执行程序

------

## 一、用户角色需求

系统至少应支持以下角色：

管理员：负责设置班级、学生、教师、课程类型、查看全部统计、导出数据。

上课老师：负责查看自己当天课程、进行签到、补签、备注学生情况。

前台或教务：负责排课、修改时间段、查看缺勤、导出统计。

家长/学生端可以暂不做，第一版建议只做内部管理端。

------

## 二、基础信息管理

### 1. 学生管理

需要支持添加学生信息。

学生信息建议包括：

姓名、性别、年龄、联系电话、家长姓名、家长电话、就读学校、备注、是否在读。

需要支持：

添加学生
修改学生信息
删除或停用学生
按姓名搜索学生
按班级筛选学生
查看学生历史签到记录
查看学生剩余课时或累计课时

删除学生时建议不要真实删除，而是设置为“已停用”，避免影响历史统计。

------

### 2. 教师管理

需要支持添加上课老师。

教师信息包括：

教师姓名、手机号、任教方向、状态、备注。

需要支持：

添加老师
修改老师信息
停用老师
查看老师授课记录
按老师统计上课课时

------

### 3. 班级管理

班级是系统核心。

班级信息建议包括：

班级名称
所属课程，例如 Scratch、Python、C++、机器人、AI 启蒙
班级类型，例如常规班、试听班、集训班、一对一、小组课
默认上课老师
默认上课时间段
班级容量
开班日期
结课日期
班级状态：未开班、上课中、已结课、暂停
备注

班级功能包括：

新建班级
编辑班级
删除或归档班级
添加学生到班级
从班级移除学生
查看班级学生名单
查看班级签到历史
查看班级课时统计

一个学生应允许加入多个班级，例如一个孩子同时参加 Scratch 和机器人。

------

### 4. 上课类型管理

系统应支持自定义上课类型。

建议默认包括：

常规课
补课
试听课
公开课
一对一
集训课
比赛辅导课
机器人课
AI 启蒙课
测评课

每种上课类型可以设置是否计入课时统计。

例如：

常规课：计入课时
补课：计入课时
试听课：可选择不计入
公开课：可选择不计入
测评课：可选择不计入

这样后期统计更准确。

## 四、签到管理

### 1. 班级签到

老师进入某个课次后，应看到该班级学生名单，然后逐个签到。

签到状态建议包括：

已到
迟到
请假
缺勤
补签
试听
停课
未确认

每个学生的签到记录应包括：

学生姓名
所在班级
课次日期
签到状态
签到时间
操作人
备注
是否计入课时

------

### 2. 快速签到

为了适合课堂使用，应支持：

一键全员已到
单个学生修改状态
批量标记请假
批量标记缺勤
课后补签
签到备注

例如老师可以先点击“一键全员已到”，再把没来的学生改成“请假”或“缺勤”。

------

### 3. 课时扣减规则

建议系统支持灵活课时规则。

例如：

已到：扣 1 次课
迟到：扣 1 次课
请假：不扣课时或按设置扣课时
缺勤：可选择扣课时
试听：不扣课时
补课：扣课时或不扣课时

为了避免后期混乱，建议在系统设置中加入“课时计算规则”。

例如：

常规课每次默认 1 课时
90 分钟课程 = 1 课时
120 分钟课程 = 2 课时
一对一课程可单独设置课时数量
试听课默认不计入正式课时

这是系统重点。

### 1. 按日统计

按某一天统计所有上课情况。

统计内容包括：

当天上课班级
当天授课老师

请假人数
缺勤人数
迟到人数
试听人数

各班级出勤率

支持按日期筛选，例如：

2026-05-14 当日统计。

------

### 2. 按周统计

按自然周或自定义日期范围统计。

统计内容包括：

本周各老师上课次数

本周学生出勤情况
本周请假名单
本周缺勤名单

支持选择：

本周
上周
指定开始日期至结束日期

### 3. 按月统计

按月份统计。

统计内容包括：

本月各班级上课次数
本月各老师授课课时

## 九、权限与安全需求

局域网系统虽然不对公网开放，但仍建议有登录功能。

至少支持：

账号登录
不同角色权限
修改密码
操作日志

权限建议：

管理员：全部权限
教师：只能查看和签到自己负责的课程
教务：可排课、统计、导出
只读账号：只能查看统计，不能修改数据

操作日志建议记录：

谁在什么时间修改了课程
谁进行了补签
谁删除或停用了学生
谁导出了数据

## 十、数据备份需求

本地系统最怕数据丢失，必须设计备份功能。

建议支持：

手动备份数据库

备份文件导出

SQLite 数据库可以备份为一个 `.db` 文件，也可以定期复制到指定文件夹。

## 十一、异常处理需求

系统需要考虑实际使用中的问题。

例如：

老师忘记签到，可以补签
学生中途加入班级，可以从加入日期开始统计
学生转班，历史记录仍保留
课程取消，不计入课时
老师临时代课，课次老师可以单独修改
学生请假后补课，需要能标记补课记录
同一天同一个学生可能有两节不同课程，应分别记录
误操作删除数据，应尽量用“停用”而不是真删除

可提前设置课程大类、阶段、第几节课和具体课程名称，例如：

课程大类
阶段

### 12.5 Vue 表单校验与账号自助维护

Vue 管理端已补充表单校验：

学生、教师、班级、课程类型、预设课程、课次、账号等弹窗表单支持必填校验。
手机号字段支持中国大陆手机号和带区号座机格式。
课时数必须大于 0，学生购买/赠送课时不能小于 0。
班级日期范围和上课时间范围会校验结束值不能早于开始值。

账号登录后可在顶部点击“修改密码”，输入当前密码、新密码和确认密码完成自助改密。

### 12.6 老师课次权限

老师账号必须关联教师资料。老师登录后：

学生、班级、课次、首页今日课次、未来课次和统计数据都会限定到自己关联教师。
老师只能查看自己名下班级的学员，直接访问其他老师班级学生名单会返回无权限。
老师可以进入本人课次进行签到、补签、填写每节课课程详情和导出本人课次签到表。
